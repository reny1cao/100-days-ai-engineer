## 计划说明

- **时长：** 全职 10 个冲刺。
- **结构：** 每个冲刺聚焦 1–2 个主题，包含目标、重点概念、推荐资料、练习任务与复盘提示。
- **输出：** 夯实 DSA、系统设计（含 LLM/大数据）、ML/LLM 理论、RAG、Agent 编排、数据工程与面试掌控能力。
- **融会贯通：** 持续关注主题之间的联动，例如数据结构支撑系统架构、ML 基础决定 LLM 系统取舍。

## 里程碑与检查点

| 冲刺 | 关键里程碑 | 建议分享 |
| --- | --- | --- |
| 1 | 完成核心数据结构练习及 CLI 工具 | 总结疑难点与复杂度分析 |
| 2 | 至少解决 3–4 道中等题并归类模式 | 录制/复述一道题的思路讲解 |
| 3 | 完成系统设计 + 大数据管道演练 | 与同伴评审设计方案与瓶颈 |
| 4 | 能清晰解释 Transformer、Prompt、RLHF | 制作术语闪卡并自测 |
| 5 | 产出 RAG 架构与评估指标草案 | 讨论向量库、切片策略取舍 |
| 6 | 演示工具型 Agent 与守护机制 | 梳理失败恢复与评估策略 |
| 7 | Demo 批流一体 ETL，展示数据质量指标 | 分享治理/成本优化经验 |
| 8 | 发布 LLM 服务 + SLO 仪表板 | 回顾混沌/可靠性测试结果 |
| 9 | 完成 75 分钟系统设计模拟 | 汇总反馈与改进计划 |
| 10 | 结束全套模拟与谈判准备 | 演练面试日流程与细节 |

## 支持与协作

- 组织或加入每个冲刺 30 分钟站会，分享进度、卡点、资源。
- 建立一份活页笔记记录反思与问题，用于第 10 冲刺回顾。
- 至少每个冲刺进行一次结对编程或系统设计演练，模拟真实合作场景。

:::caution[守护专注时间]
提前在日历中锁定学习时段，避免碎片化任务打断。不必追求超长时间，持续稳定最重要。
:::

## 快速开始

1. 阅读 **[总览](./guides/interview-prep/overview)**，了解每日节奏与复盘模板。
2. 预留每个冲刺的固定时间块（算法、系统设计、行为模拟等）。
3. 建立任务/习惯清单，记录当冲刺的任务与反思。
4. 若有社区渠道（Slack/Discord 等），及时同步心得与问题。

祝备考顺利！从当前冲刺开始行动，坚持输出与复盘，你会在十个冲刺内建立系统的面试思维与自信。
